# Chunking Strategy

This document describes how files are split into chunks for embedding and search.

## Per-File-Type Strategies

### PDF / DOCX / PPTX (Paginated Documents)

Source: `[Page: N]` tags generated by Gemini during content extraction.

- Chunks are created per page using `PageDelimiters` from position data
- Each chunk's `PageRange` maps back to the original page number
- Long pages are sub-split at `maxChunkChars` (60,000) for Milvus varchar safety

### Video / Audio (Timestamp-Based)

Source: `[Video: HH:MM:SS]`, `[Audio: HH:MM:SS]`, `[Sound: HH:MM:SS]` markers in transcripts.

- `injectMediaPageDelimiters()` scans for timestamp markers and creates synthetic `PageDelimiters` at ~4000 char intervals, snapping to the nearest timestamp
- `populateTimeRanges()` sets `TimeRange` on each chunk for time-based citations
- **Target**: ~1000 tokens (4000 chars) per chunk
- **Before**: 60,000 char max created 2-3 chunks for a 50-minute video. **After**: 40-50 chunks

### XLSX / XLS (Sheet-Based)

Source: `[Sheet: SheetName]` tags generated by `convertSpreadsheetToMarkdown` using `excelize`.

- XLSX files are parsed directly; XLS files are first converted to XLSX via LibreOffice
- Each sheet becomes a section delimited by `[Sheet: SheetName]` markers
- Content is formatted as markdown tables with header/separator/data rows
- Rows are capped at `maxSpreadsheetRows` (10,000) and columns at `maxSpreadsheetCols` (50) with truncation notes
- Pipe characters and newlines in cell values are escaped for markdown table safety
- Citations use `UNIT_SHEET` with the sheet name as `HighlightText`

### Markdown / Plain Text (Heading-Based)

Source: Markdown heading patterns (`# `, `## `, `### `, `#### `) and paragraph breaks (`\n\n`).

- `injectMarkdownPageDelimiters()` creates boundaries at ~4000 char intervals, snapping to the nearest heading or paragraph break
- Falls back to paragraph breaks when no headings exist
- Content shorter than `markdownChunkTarget` (4000) is kept as a single chunk

### Images

Images use `TYPE_CONTENT` chunks from Gemini's OCR/description output plus `TYPE_SUMMARY` and `TYPE_AUGMENTED` chunks for conceptual search.

## Chunk Types

| Type | Purpose | Source |
|------|---------|--------|
| `TYPE_CONTENT` | Raw extracted content for citation | Content extraction activity |
| `TYPE_SUMMARY` | Prose summary (<500 words) | ProcessSummaryActivity |
| `TYPE_AUGMENTED` | Entity/topic tags for search | ProcessSummaryActivity (entities section) |

## Constants

| Constant | Value | Purpose |
|----------|-------|---------|
| `maxChunkChars` | 60,000 | Milvus varchar safety limit |
| `mediaChunkTarget` | 4,000 | Target chunk size for video/audio |
| `markdownChunkTarget` | 4,000 | Target chunk size for markdown/text |
